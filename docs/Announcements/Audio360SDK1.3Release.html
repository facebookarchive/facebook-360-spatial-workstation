Audio360 v1.3 Release¶

v1.3 of the Audio360 SDK is now available. v1.x of the SDK has been in wide use across the Facebook and Oculus ecosystem and by many of our partners. For those upgrading from 0.9.x, v1.3 is a major rewrite with a focus on improved performance, cross-platform integrations and wider platform support. We recommend upgrading to v1.3 as support for 0.9.x will shortly be deprecated.

Partner Apps¶

The Audio360 SDK powers many of our partner apps, including Within, Lytro (with 6DOF tracking), JauntVR and Felix And Paul Studios. Mixes from Spatial Workstation can be deployed to Within, with support in JauntVR rolling out soon.

What’s New¶

A New API¶

  • A new cross-platform C++ API for all native integrations. The older C API is no longer available, but you should find much of the syntax and functionality to be similar
  • A new and improved Android Java API, with support for event callbacks and offering full parity with the native API
  • A new Unity integration (currently in preview) with support for event callbacks and improved .obb support

New Integrations¶

  • Integration for Android’s ExoPlayer (v2.2, support for newer versions to follow) for easy streaming and playback of videos with spatial audio using new format now available in the FB360 Encoder — “MKV-Opus”. Spatial Workstation mixes are compressed into opus and muxed into MKV containers. The ExoPlayer integration makes it possible to playback such files on Android with just a few lines of code.
  • Example integration with libFFmpeg. The example shows how FFmpeg can be used with the Audio360 SDK to playback the MKV-Opus file created with the FB360 Encoder.

Lower-Level Access, New Formats¶

The SDK continues to support playback of .tbe files, but can also playback .wav files in a variety of formats and configurations — First Order ambiX, TBE/HHOA 8 channel, Second Order ambiX — each with and without head-locked audio.

It also offers lower-level access to:

  • Use custom IO stream objects
  • Enqueue and spatialise streams of ambisonic PCM audio
  • Decode .wav/tbe spatial audio files or opus packets to PCM
  • Optimise streaming buffer sizes and memory allocation across the engine

Wider Platform Support, Improved Performance¶

In addition to existing support for Windows 32/64 bit, macOS, iOS and Android (armeabi-v7a), the SDK now supports:

  • Windows UWP (for Windows Mixed Reality)
  • Android arm64-v8a
  • Android x86
  • Android x86_64

With up to 4x improvements in performance, the SDK also features fast in-cache seeking, support for efficiently mixing a large number of objects and improved thread-safety.

Upgrading¶

If you are currently using 0.9.x of the SDK, upgrading to 1.3 and greater will require changes.

Native integrations:

  • The binary name has changed from TBAudioEngine to Audio360
  • The C API is no longer available, all API calls will have to be changed to the C++ interface

Java:

  • APIs are similar, but are available under the audio360.* java package

Unity:

  • API functionality is similar, but we do not recommend importing 0.9.x and 1.3 into the same project
  • All new APIs are under a TBE namespace
  • A listener script (TBEngineListener.cs in 0.9.x) is no longer required
  • TBSpatDecoder is now named TBE.SpatDecoderFile

Please see the user guide for platform specific considerations.
